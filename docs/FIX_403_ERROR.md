# Исправление ошибки 403 в HeadHunter парсере

## Проблема
HeadHunter API возвращал ошибку 403 (Forbidden) при попытке получить полное описание вакансий.

## Решение
Внесены следующие изменения в `parsers/headhunter_parser.py`:

### 1. Улучшены HTTP заголовки
Добавлены дополнительные заголовки для более корректной работы с API:
- `Accept-Encoding: gzip, deflate, br`
- `Connection: keep-alive`
- `Sec-Fetch-*` заголовки

### 2. Увеличена задержка между запросами
- Было: `0.2` секунды
- Стало: `0.5` секунды

Это снижает вероятность блокировки из-за слишком частых запросов.

### 3. Изменена стратегия парсинга (главное изменение)

**Новая стратегия:**
1. **Сначала используем данные из списка** - они уже есть в ответе API и не требуют дополнительных запросов
2. **Опционально запрашиваем полное описание** - только если нужно больше информации
3. **При 403 ошибке** - спокойно используем данные из списка (это нормально)

**Преимущества:**
- ✅ Меньше запросов к API = меньше 403 ошибок
- ✅ Быстрее парсинг (не нужно ждать каждый запрос)
- ✅ Надежнее (даже при 403 получаем вакансии)
- ✅ Основная информация (название, компания, зарплата, ссылка) всегда доступна

### 4. Улучшена обработка ошибок
- 403 ошибки теперь обрабатываются как нормальная ситуация (используем данные из списка)
- Добавлены `log.debug()` вместо `log.warning()` для 403, так как это не критично
- Улучшена обработка других ошибок

## Результат

Теперь парсер:
- ✅ Работает даже при 403 ошибках
- ✅ Получает вакансии быстрее
- ✅ Меньше нагружает API
- ✅ Более надежен

## Что изменилось для пользователя

**Ничего!** Парсер работает так же, но теперь:
- Меньше предупреждений в логах (403 обрабатываются тихо)
- Быстрее получает вакансии
- Более стабильно работает

## Технические детали

### Логирование
- `log.debug()` - для 403 ошибок (не критично, используем данные из списка)
- `log.warning()` - для других ошибок (когда действительно проблема)

### Задержки
- `0.3` секунды - перед запросом полного описания
- `0.5` секунды - между обработкой вакансий
- `1` секунда - между страницами

### Fallback механизм
Если полное описание недоступно (403 или другая ошибка), используется краткое описание из списка, которое содержит:
- Название вакансии
- Компанию
- Зарплату
- Краткое описание (snippet)
- Ссылку на вакансию

Это достаточно для большинства случаев использования.
